# План: настоящий навигатор в режиме вождения

Документ — внутренний план задачи. Когда соберёшься делать, открой его и иди сверху вниз.

---

## Цель

Превратить «режим вождения» из карты с прямой линией в полноценный пошаговый навигатор уровня 2GIS / Яндекс.Карт:

- Маршрут идёт по реальным дорогам Питера, с учётом мостов, односторонних, проездов.
- На карте видно, **где свернуть** и **куда завернуть** — стрелки манёвров, активный сегмент пути.
- Голос на русском подсказывает: «через 200 метров поверните направо на Бухарестскую».
- Карта поворачивается по направлению движения, точка едет вдоль маршрута.
- Если съехал с маршрута — он перестраивается автоматически.
- Когда подъезжаешь к адресу — открывается кнопка «✓ доставлено».

## Этапы работы

### Этап 1. Маршруты по реальным дорогам

**Что делаем:**
- Поднимаем OSRM-сервер с данными по СЗФО России (docker-образ `osrm-backend`, регион качается с Geofabrik, импорт один раз).
- Создаём модуль `lib/routing.ts` с функциями:
  - `getRoute(points: LatLng[])` — возвращает геометрию маршрута, общее расстояние, время и список шагов с инструкциями.
  - `getDistanceMatrix(points: LatLng[])` — матрица «каждый-к-каждому» для оптимизации порядка объезда.
- Заменяем расчёт по прямой (`haversineKm`) на реальные расстояния.
- Алгоритм 2-opt прогоняется уже по дорожной матрице — порядок объезда становится честно оптимальным (учитываются мосты через Неву и т. д.).

**Готово, когда:**
- На карте между точками идёт линия по улицам, а не по прямой.
- Расстояние и ETA в боковой панели совпадают с тем, что показывает 2GIS, в пределах 5–10%.
- Тест: 10 реальных адресов из истории курьера → сравнение с Яндекс/2GIS.

**Файлы:**
- `artifacts/salary-calendar/src/lib/routing.ts` (новый).
- `artifacts/salary-calendar/src/lib/route-optimizer.ts` (правка: использует матрицу).
- `artifacts/salary-calendar/src/pages/MapView.tsx` (правка: подтягивает реальный маршрут).

### Этап 2. Отрисовка маршрута и активного сегмента

**Что делаем:**
- Рисуем полную линию маршрута приглушённо.
- Активный сегмент «от тебя до следующей точки» — ярче и толще.
- Пройденные участки — серым.
- На каждом ключевом повороте маркер-стрелка (влево / вправо / прямо / разворот).
- Опционально: поворот карты в направление движения (см. решение 4).

**Готово, когда:**
- Открываешь режим вождения — видишь не линию-крестик, а реальный путь по дорогам.
- При движении пройденная часть тускнеет, активная подсвечивается.

**Файлы:**
- `artifacts/salary-calendar/src/components/DeliveryMap.tsx` (правка).
- `artifacts/salary-calendar/src/components/DriveMode.tsx` (правка).

---

### Этап 3. Отслеживание движения по маршруту

**Что делаем:**
- На каждое обновление GPS считаем проекцию позиции на ближайший участок маршрута.
- Знаем: текущий шаг, метров до следующего поворота, метров до конца сегмента, метров до конца всего маршрута.
- Когда подъезжаешь к повороту ближе чем 250 м / 80 м / 30 м — срабатывают подсказки.
- При прибытии на точку (расстояние < 30 м, скорость близка к 0) — автоматически открываем диалог «✓ доставлено».

**Готово, когда:**
- Сверху на экране всегда видно: «через X метров — направо на улицу Y».
- При проезде поворота карточка манёвра обновляется на следующий.

**Файлы:**
- `artifacts/salary-calendar/src/lib/route-progress.ts` (новый, чистая логика).
- `artifacts/salary-calendar/src/components/DriveMode.tsx` (правка).

### Этап 4. Голосовые подсказки

**Что делаем:**
- Используем встроенный в браузер `speechSynthesis` (Web Speech API) с русским голосом.
- Шаблоны на русском пишем сами: «через 200 метров поверните направо на Бухарестскую улицу», «поворачиваем», «прибыли».
- Кнопка mute в шапке режима вождения.
- Проверяем работу на iPad — Safari блокирует автозапуск речи без жеста, но «поехали» — это и есть жест, проблем быть не должно.

**Готово, когда:**
- Едешь без рук — голос ведёт по маршруту, как настоящий навигатор.

**Файлы:**
- `artifacts/salary-calendar/src/lib/voice.ts` (новый).
- `artifacts/salary-calendar/src/components/DriveMode.tsx` (правка).

### Этап 5. Перестроение при съезде с маршрута

**Что делаем:**
- Если проекция твоей позиции уехала дальше 60 м от линии маршрута — считаем, что съехал.
- Запускаем повторный запрос к OSRM от текущей позиции до следующей точки.
- Показываем короткое уведомление «маршрут перестроен», заменяем активную геометрию.
- Защита от дёрганья: не перестраивать чаще раз в 10 секунд и не на первых 5 секундах после старта.

**Готово, когда:**
- Сворачиваешь не туда — через секунду на карте появляется новый маршрут от твоей позиции, голос объявляет следующий поворот.

**Файлы:**
- `artifacts/salary-calendar/src/lib/route-progress.ts` (правка).

### Этап 6. UI режима вождения

**Что делаем:**
- Сверху — большая карточка манёвра: иконка-стрелка (влево/вправо/прямо), расстояние крупно, название улицы.
- Под ней — следующий манёвр мелким шрифтом («потом — прямо 1.2 км»).
- Снизу — итог по маршруту: осталось точек / км / минут.
- Над итогом — текущая остановка + крупные кнопки «доставлено / пропустить».
- Тёмная тема для ночной езды по умолчанию.
- Опциональная кнопка «открыть в Яндекс.Картах / 2GIS» для текущей точки — на случай если водитель хочет сверить.
- Все элементы под палец, контрастные, читаются на солнце.

**Готово, когда:**
- На планшете в машине всё читается с полуметра, попадаешь по кнопкам без прицеливания.

**Файлы:**
- `artifacts/salary-calendar/src/components/DriveMode.tsx` (большая правка).

### Этап 7. Полевые тесты и доводка

**Что делаем:**
- Прогон 10–15 реальных адресов из истории.
- Сравнение с 2GIS / Яндексом по расстоянию и времени (допустимо 5–10% отклонение).
- Реальная езда на планшете, фиксация косяков (что не успеваешь увидеть, где голос путается, где маршрут странный).
- Доводка по результатам.

**Готово, когда:**
- Курьер реально пользуется только этим навигатором за смену и не открывает 2GIS.

## Что **не входит** в первую версию (намеренно)

Чтобы не утонуть, осознанно откладываем:

- **Пробки в реальном времени** — нет в бесплатных API. Только Яндекс/2GIS платно.
- **Подсказки по полосам** («перестройтесь во вторую правую») — есть только в нативных SDK.
- **Камеры и ограничения скорости** — нужен отдельный источник данных.
- **3D-вид зданий** — переезд на MapLibre/Mapbox + работа со стилем.
- **Оффлайн-режим** — карты тянутся тайлами по сети. Решаемо отдельным шагом (предзагрузка тайлов).

Каждое из этого — отдельная задача после того, как базовый навигатор заработает.

## Риски и подводные камни

- **OSM-данные могут устареть** — если в Питере недавно изменили схему движения или открыли новую дорогу, OSRM узнает об этом только после следующего обновления (раз в неделю-месяц).
- **Развод мостов** — статически OSM это знает (теги в дорогах), но «сейчас мост разведён» — в реальном времени никто бесплатно не отдаёт.
- **iPad / Safari** — `speechSynthesis` иногда обрывает длинные фразы. Решение: дробить подсказки на короткие.
- **Высокий расход батареи** — постоянный GPS + анимация карты + голос. Планшет в машине должен быть на зарядке всю смену.
- **Точность GPS в центре** — высокие здания дают «дрейф» в 20–50 м. Это не наша проблема, но влияет на ощущение «навигатор тупит».

## Глоссарий

- **OSRM** — Open Source Routing Machine, движок маршрутизации поверх данных OpenStreetMap. Открытый и бесплатный.
- **OpenStreetMap (OSM)** — свободная мировая карта, аналог Википедии, но для геоданных. Качество зависит от района.
- **MapLibre GL** — открытый форк Mapbox GL, рендерит карту через WebGL, умеет поворот и наклон из коробки.
- **2-opt** — алгоритм для задачи коммивояжёра: перебирает пары участков маршрута и переворачивает их, если становится короче.
- **Web Speech API** — встроенный в браузер синтез речи. На большинстве систем есть русский голос.
- **Capacitor** — обёртка, превращающая веб-приложение в нативное приложение для iOS/Android.
